callable接口类似于Runnable, 因为它们都是为其实例可能由另一条线程执行的类设定的.
Runnable 不返回结果, 也不能抛出被检查的异常 因此有了callable

callable 特点

  • 可以有返回值
  • 可以抛出异常
  • 方法不同, run() / call()
    例子: 泛型的参数等于方法的返回值

细节

  • 有缓存
  • 结果可能需要等待, 会阻塞!
  • 上述例子 只打印一次是因为FutureTask只会被线程执行run 一次
