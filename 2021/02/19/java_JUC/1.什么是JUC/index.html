什么是JUC

JUC:

  • java.util.concurrent
  • java.util.concurrent.atomic
  • java.util.concurrent.locks

基础多线程中的 callable, lock锁就是属于juc中的内容

线程和进程

线程,进程, 如果不能用一句话说出来的技术, 不扎实!
进程

  • 一个程序, 程序的集合
  • 一个进程往往包含多个线程, 至少包含一个!

java默认有几个线程? 两个

  • main
  • GC 垃圾回收

线程

  • 开了一个进程, 写字, 自动保存 (线程负责)
  • Thread Runnable Callable 三种开启线程的方式

java真的可以开启线程吗?

并发与并行

并发编程: 并发, 并行
并发: 多线程操作同一个资源

  • CPU 一核, 模拟出来多条线程, 天下武功唯快不破, 快速交替
  • 两个人同时在一个账户上取钱

并行: 多个人一起行走

  • 多核CPU下, 多个线程可以同时执行
  • 两个人同时取钱, 分别在不同的账户上

获取cpu核数

并发编程的本质: 充分利用CPU的资源

线程有几个状态

  • 创建 NEW
  • 运行 RUNNABLE
  • 阻塞 BLOCKED
  • 等待 WAITING
  • 超时等待 TIMED_WAITING 过期不候
  • 死亡 TERMINATED

wait/sleep的区别

  • 来自不同的类
    • wait —> object
    • sleep —> Thread
  • 锁的释放
    • wait 会释放锁
    • sleep 不会 抱着锁睡觉
  • 使用的范围不同
    • wait 必须在同步代码块中
    • sleep 可以在任何地方谁
