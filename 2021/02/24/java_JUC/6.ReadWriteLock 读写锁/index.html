6.ReadWriteLock 读写锁
TZ Lv5
 2021-02-24 21:33:00  
  • java 
 
  • JUC 
 720 Words  3 Mins

A ReadWriteLock维护一对关联的locks ，一个用于只读操作，一个用于写入。 read lock可以由多个阅读器线程同时进行，只要没有作者。 write lock是独家的。
读的时候可以多个人去读, 写只能由一个线程去写
不使用ReadWriteLock 进行读写分离的话会出现的错误如下:

正确的方法应该是:

readWriteLoad

  • 读-读 可以共存!
  • 读-写 不能共存!
  • 写-写 不能共存!

独占锁(写锁) 一次只能被一个线程占有
共享锁(读锁) 多个线程同时占有

为什么读的时候也要加锁?
因为读的过程中也有可能出现写入的情况,所以加锁,读完再写
