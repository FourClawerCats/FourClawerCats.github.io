线程池的好处

  • 降低资源的消耗
  • 提高相应的速度
  • 方便管理
  • 线程可以复用
  • 控制并发数
  • 管理线程

线程池 三大方法, 七大参数, 四种拒绝策略

三大方法

Executors

Executors.newSingleThreadExecutor(); 单个线程

Executors.newFixedThreadPool(5); 创建一个固定的线程池的大小

Executors.newCachedThreadPool(); 可伸缩的, 遇强则强,遇弱则弱

七大参数

Executors.newSingleThreadExecutor() 源码

Executors.newFixedThreadPool(5) 源码

Executors.newCachedThreadPool() 源码

三大方法本质都是调用了 ThreadPoolExecutor;

ThreadPoolExecutor 源码

ThreadPoolExecutor 有七个参数, 就是我们说的 三大方法, 七个参数 中的参数

ThreadPoolExecutor 七个参数

  • int corePoolSize 核心线程池大小
    int maximumPoolSize, 最大线程池大小
    long keepAliveTime, 超时了 没有人调用就会释放
    TimeUnit unit, 超时单位
    BlockingQueue workQueue 阻塞队列
    ThreadFactory threadFactory, 线程工厂 创建现成的, 一般不用动
    RejectedExecutionHandler handler 拒绝策略

手动创建一个线程池

RejectedExecutionHandler 的四个实现类, 四种拒绝策略

AbortPolicy 默认策略 银行满了, 还有人进来, 不处理这个人, 抛出异常

CallerRunsPolicy 哪来的 回哪里去 打发了

DiscardOldestPolicy 队列满了不会抛出异常 丢掉任务, 不会抛出异常

DiscardPolicy 队列满了, 尝试和第一个人竞争 也不会抛出异常

线程池的最大值, 应该如何去设置!

  • CPU密集型
    • 如cpu是12核, 12条线程同时执行!
    • 几核就是几, 可以保证cpu的效率最高!
    • java获取cpu的核心数: Runtime.getRuntime().availableProcessors()
  • IO密集型
    • 判断你的程序中十分消耗线程的io, 大于你的io消耗线程即可, 一般是两倍
    • 一个程序 有15个大型任务 io十分占用资源! 这时候就把线程设置为30
