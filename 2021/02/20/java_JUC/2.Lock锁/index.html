传统synchronized 和lock 实现的区别

synchronized例子

lock例子:

  • Synchronized 内置的Java关键字, lock是一个java类
  • Syschronized 无法判断获取锁的状态, Lock可以判断是否获得了锁
  • Synchronized 会自动释放锁, lock必须手动释放锁! 如果不释放, 死锁
  • Synchronized 线程一 获得锁,阻塞, 线程2 一直等待
  • lock锁 不会傻傻的等待, 可以用trylock() 尝试获取锁
  • Synchronized 可重入锁, 不可中断, 非公平,
  • Lock 可重入锁, 可以判断锁, 非公平 公平 可以自己设置 ReentrantLock() true 公平 默认不公平 false
  • Synchronized 适合锁少量的代码同步问题
  • lock 适合锁大量的同步代码
